## üõ† Contributing a New Project

### Step 1: Define the Project

1. Create an Issue describing:
   - Project name and purpose
   - Technology stack (Python, C++, etc.)
   - Key features and requirements
   - Success criteria
   - Estimated complexity

2. Wait for feedback and approval from maintainers

### Step 2: AI Implementation

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/project-name`
3. **Use an AI agent to write all code**
   - Specify requirements clearly
   - Review AI-generated code for quality
   - Iterate on improvements
   - Document the AI tool used
4. Create project folder: `projects/[project-name]/`
5. Add source code and supporting files
6. Add comprehensive README.md

### Step 3: Compliance Check

Before submitting PR, ensure:

- ‚úÖ ALL code is AI-written (no human code)
- ‚úÖ Project follows folder structure
- ‚úÖ README.md is comprehensive
- ‚úÖ Code is functional and tested
- ‚úÖ Compliance marks are included
- ‚úÖ AI tool attribution is documented

### Step 4: Submit Pull Request

1. Push to your fork
2. Create Pull Request with:
   - **Title**: Descriptive project name
   - **Description**:
     ```
     ## Project: [Name]
     
     **AI Tool Used**: [Copilot/ChatGPT/Claude/Other]
     
     **Description**: Brief project overview
     
     **Features**:
     - Feature 1
     - Feature 2
     - Feature 3
     
     **Technology**: Python/C++/etc
     
     **AI Compliance**: ‚úÖ All code written by AI agent
     ```
3. Await review and feedback
4. Iterate based on comments

## üëç Code Review Guidelines

When reviewing AI-written code, consider:

### Code Quality
- Is the code readable and maintainable?
- Are function/variable names clear?
- Is there appropriate error handling?
- Are there code comments explaining complex logic?

### Architecture
- Does it follow design patterns appropriately?
- Is the module organization logical?
- Are dependencies well-managed?
- Could it scale if needed?

### Security
- Are input validations present?
- Are there potential buffer overflows (C++)?
- Is network communication secure?
- Are credentials/secrets handled safely?

### Performance
- Are algorithms efficient?
- Could performance be improved?
- Are resources properly managed?
- Is memory usage reasonable?

### Testing
- Has it been tested thoroughly?
- Do edge cases work correctly?
- Are error conditions handled?
- Does it match the specification?
